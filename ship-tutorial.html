PATH 설정 echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.zshrc source ~/.zshrc which ship # → /Users/[이름]/.local/bin/ship

Git 워크플로 동작 방식

ship은 실행 시작부터 끝까지 git 작업을 자동으로 처리합니다.

  • 1
    브랜치 자동 생성 — main·master·prd·develop에서 실행하면 feat/{slug}-{YYYYMMDD-HHmmss} 브랜치를 자동 생성. 이미 feature 브랜치에 있으면 현재 브랜치 사용.
  • 2
    중간 커밋/푸시 — Wave A 완료 후, Wave B 완료 후, 자동 수정 라운드마다 자동 커밋. 변경사항 없으면 스킵.
  • 3
    PR 생성 (항상 실행) — CI 게이트 통과 여부와 무관하게 Phase 6에서 항상 PR을 생성. 게이트 실패 시 Draft PR, 통과 시 일반 PR. 라벨 Claude Agent 자동 부착.
🚫
PR 베이스 브랜치 보호
릴리스 매니저는 prd·master를 베이스로 한 PR을 절대 생성하지 않습니다. 해당 상황이 감지되면 즉시 중단하고 사유를 보고합니다.

문제 해결

ship: command not found

export PATH="$HOME/.local/bin:$PATH" which ship

permission denied: ship

chmod +x ~/.local/bin/ship

tmux 패널이 분할 안 됨

echo $TMUX # 비어있으면 tmux 밖에서 실행 중 tmux new-session -s work

PR이 생성되지 않음

# gh 인증 확인 gh auth status # 릴리스 매니저 로그 확인 cat .agent/ship-[타임스탬프]/release_manager.log
ℹ️
Phase 6은 게이트 실패 시에도 항상 실행됩니다. PR이 생성되지 않았다면 release_manager.log에서 원인을 확인하세요.

에이전트가 항상 기본값으로 나옴

# agents.txt 확인 cat .agent/ship-[타임스탬프]/agents.txt # architect 로그 확인 cat .agent/ship-[타임스탬프]/architect.log

토론이 너무 오래 걸림

Debate는 에이전트 3개(Red/Blue/Judge)가 순차 실행되므로 라운드당 시간이 걸려요. ship 파일에서 run_debate 함수의 MAX_ROUNDS=2를 1로 줄이면 빨라져요.

토론 결과 확인

# 가장 최근 실행 로그 목록 ls -t .agent/ | head -1 | xargs -I{} ls .agent/{} # 판정 결과만 보기 cat .agent/ship-[타임스탬프]/judge_plan_1.md cat .agent/ship-[타임스탬프]/judge_review_1.md
